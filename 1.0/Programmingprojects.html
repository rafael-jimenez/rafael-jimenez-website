Federal Skilled Worker Program Eligibility Evaluator in Java


Project Description

This project focused on developing a Java application to process and evaluate applicants for the Federal Skilled Worker Program. The goal was to apply fundamental programming concepts, including decision-making, loops, functions, and file I/O, to create an application that reads data from an input file, processes the applicants' information, and outputs a list of qualified candidates based on a simplified version of the Express Entry scoring system.

Key Learnings and Insights:
  • Transformed basic Python programming constructs (such as loops, decision-making, and file handling) into Java, gaining proficiency in Java syntax and object-oriented programming concepts.
  • Applied modular programming by organizing code into functions to ensure better maintainability, readability, and reusability.
  • Gained practical experience in file I/O by reading data from a tab-separated input file and writing qualified applicants' information to an output file.
  • Implemented a scoring system based on multiple factors, including language skills, education, work experience, age, and adaptability, to determine applicants' eligibility for the program.
  • Refined debugging skills by testing the application with multiple datasets and ensuring the program handled different cases and edge conditions effectively.



Casino Management System in Java


Project Description

This project focused on developing a Java application for the Punto Banco casino game, incorporating features such as player management, balance tracking, and game statistics. The goal was to apply object-oriented programming principles to implement the game's rules, handle player data, and allow users to play, place bets, and view player rankings. The application also involves storing player data in a file and providing reports on top players and individual player information.

Key Learnings and Insights:
  • Enhanced object-oriented programming skills by creating classes for cards, players, and decks, and implementing core game mechanics for Punto Banco.
  • Gained experience in data persistence by implementing file I/O to store and retrieve player information, including balances and wins, from a text file.
  • Developed a robust menu system with user input validation, allowing for smooth navigation and interaction with the game.
  • Implemented game logic for Blackjack, handling player bets, card dealing, and determining the outcome based on game rules, including blackjack and dealer strategies.
  • Applied JUnit testing to validate the correctness of key components, such as card creation, hand evaluation, and deck shuffling, ensuring reliable gameplay.
  • Refined debugging skills by testing the application with various datasets and ensuring the game logic and player management worked correctly in all cases.



Toy Store Inventory Management System in Java


Project Description

This project focused on developing a Java-based inventory management system for a toy store, enabling users to find, purchase, and manage toys within the store's database. The application was designed using object-oriented principles such as inheritance, abstraction, and interfaces to handle different types of toys and their specific attributes. The system includes functionality to search for toys, add or remove toys, and make gift suggestions based on user input, while ensuring that data is stored and retrieved efficiently using file I/O.

Key Learnings and Insights:
  • Enhanced understanding of object-oriented programming by utilizing inheritance to create specialized classes for each toy type while maintaining shared attributes in a superclass.
  • Applied file handling techniques for storing and retrieving toy data from a text file, ensuring data persistence and integrity across application sessions.
  • Developed a user-friendly interface with input validation, enabling customers to search for toys, purchase items, and manage the inventory with ease.
  • Implemented a toy suggestion feature, allowing users to receive gift recommendations based on age, type, or price range, enhancing the user experience.
  • Refined debugging and testing skills by ensuring that the system correctly handled edge cases such as invalid inputs, stock limitations, and unique serial numbers.
  • Strengthened knowledge of abstract classes and interfaces to define common behaviors for different toy types, while providing flexibility in their implementation.
